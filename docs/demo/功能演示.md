# GoAdmin 功能演示

本文根据 `docs/demo/` 下的 4 张截图整理，按“菜单管理 → CodeGen 控制台 → 生成的 CRUD 页面 → 国际化页面”的顺序展示当前项目的主要能力。

## 演示概览

GoAdmin 当前的功能演示可以重点体现以下几类能力：

- **菜单与权限治理**：支持系统菜单、路由、组件和权限元数据的统一维护。
- **CodeGen 生成控制台**：支持 DSL / 数据库输入、Dry-run 预览、文件计划查看与一键生成。
- **CRUD 页面生成结果**：生成后的列表页遵循统一的管理后台风格，并可直接用于业务模块。
- **前端国际化**：支持中英文切换，页面文案可随语言环境同步变化。

## 1. 菜单管理

![菜单管理](./1%20preview.png)

### 演示说明

该页面展示的是系统的**菜单管理**能力，用于维护后台导航、路由、组件、权限标识等核心元数据。

### 可讲解要点

- **菜单结构清晰**：左侧为模块导航，主区域展示菜单列表。
- **支持筛选与查询**：可按名称、父级菜单等条件检索。
- **支持编辑与新增**：列表右侧可进行编辑、删除操作，右上角提供新增入口。
- **统一管理系统资源**：菜单数据不仅影响侧边栏展示，也会影响路由和权限控制。

### 演示价值

这张图适合用来说明 GoAdmin 的后台不是静态页面集合，而是一个可配置、可治理的管理平台基础。

## 2. CodeGen 控制台

![CodeGen 控制台](./2%20codegen.png)

### 演示说明

该页面展示的是 **CodeGen 控制台**，用于接收 DSL 或数据库输入，并生成模块、CRUD、权限和相关文件计划。

### 可讲解要点

- **输入方式灵活**：支持 DSL 与数据库两类入口。
- **支持 Dry-run 预览**：生成前可先查看执行结果，降低误生成风险。
- **文件计划可视化**：右侧区域展示将要创建、覆盖或追加的文件清单。
- **执行结果明确**：生成动作与预览结果分区展示，便于追踪。
- **适合模块化开发**：生成内容与项目目录结构保持一致，便于直接落地。

### 演示价值

这张图最适合用来说明 GoAdmin 的核心效率能力：**通过统一生成器快速创建业务模块，并保持前后端结构一致**。

## 3. 生成的 CRUD 页面

![生成的 CRUD 页面](./3%20created%20crud.png)

### 演示说明

该页面展示的是 CodeGen 生成后的 **图书列表 CRUD 页面**，说明生成结果已能直接进入后台使用。

### 可讲解要点

- **列表页结构统一**：包含查询区、表格区、分页区和操作区。
- **支持新增入口**：右上角提供新增按钮。
- **字段列清晰**：展示 ID、租户、书名、作者、ISBN、出版社、发布日期、分类、状态等业务字段。
- **适配后台管理场景**：页面风格与系统其他模块一致，降低学习成本。

### 演示价值

这张图可用于说明 GoAdmin 的 CodeGen 不只是“生成代码”，而是能生成**可直接接入的标准后台页面**。

## 4. 国际化页面

![国际化页面](./4%20i18n.png)

### 演示说明

该页面展示了 **国际化（i18n）** 效果，界面文案切换到了英文，证明前端可以根据语言环境展示不同语言内容。

### 可讲解要点

- **界面语言切换**：顶部语言区域已切到 English。
- **菜单与页面同步翻译**：左侧导航和主页面标题均展示英文。
- **列表与表单文案本地化**：搜索条件、按钮、表头、状态等都使用英文文案。
- **适合多语言场景**：说明系统可以面向不同语言环境部署和使用。

### 演示价值

这张图适合用来说明 GoAdmin 已具备较完整的前端国际化能力，可支撑后续更大范围的多语言扩展。

## 建议演示顺序

如果将这 4 张图用于现场讲解，建议按下面顺序展开：

1. **菜单管理**：先说明系统的基础治理能力。
2. **CodeGen 控制台**：再说明如何快速生成业务模块。
3. **CRUD 页面**：接着展示生成结果的可用性。
4. **国际化页面**：最后强调产品的可扩展性与多语言能力。

## 结语

这组截图可以完整呈现 GoAdmin 的四个重点方向：

- **平台治理能力**
- **代码生成能力**
- **标准化业务页面能力**
- **多语言国际化能力**
