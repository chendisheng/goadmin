# GoAdmin 项目功能概览

GoAdmin 是一个面向管理后台场景的模块化平台，围绕权限治理、业务模块生成、标准化页面能力和多语言支持展开，帮助开发者更快构建可维护、可扩展的后台系统。

本文结合 `docs/images/` 下的 4 张截图，展示项目的核心功能与主要使用场景。

## 核心能力

- **统一的后台治理能力**：菜单、路由、组件和权限元数据集中管理，支持后台系统的结构化配置。
- **模块化代码生成能力**：通过 CodeGen 快速生成模块、CRUD 页面和相关资源，降低重复开发成本。
- **可直接使用的业务页面**：生成结果遵循统一的管理后台规范，可直接进入业务开发流程。
- **完善的国际化支持**：前端支持中英文切换，菜单、表格、表单和按钮文案可随语言环境同步变化。
- **面向开源社区的工程组织**：目录结构清晰，强调后端模块化、前端标准化和文档可维护性。

## 功能展示

### 1. 菜单管理

![菜单管理](./images/1%20preview.png)

菜单管理页面用于维护系统导航、路由、组件和权限标识，是整个后台治理能力的基础。

**页面特点**

- 支持按名称、父级菜单等条件查询。
- 支持新增、编辑、删除等常见维护操作。
- 菜单数据会同时影响侧边栏、路由和权限控制。
- 适合作为模块化后台的统一入口管理界面。

**适用场景**

- 系统导航配置
- 模块路由编排
- 按钮级权限控制
- 后台资源结构治理

### 2. CodeGen 控制台

![CodeGen 控制台](./images/2%20codegen.png)

CodeGen 控制台是 GoAdmin 的核心效率工具，用于从 DSL 或数据库结构生成业务模块、CRUD 页面和配套文件。

**页面特点**

- 支持 DSL 与数据库两类输入方式。
- 支持 Dry-run 预览，生成前可查看执行计划。
- 文件计划可视化，便于确认将要创建、覆盖或追加的内容。
- 生成结果与预览结果分区展示，便于追踪和回溯。
- 与项目目录结构保持一致，适合直接纳入实际开发流程。

**适用场景**

- 新业务模块初始化
- 标准 CRUD 页面生成
- 权限与资源脚手架生成
- 代码结构统一化落地

### 3. 生成的 CRUD 页面

![生成的 CRUD 页面](./images/3%20created%20crud.png)

该截图展示了 CodeGen 生成后的图书列表页面，说明生成结果不仅停留在代码层面，也可以直接形成可用的后台界面。

**页面特点**

- 统一的列表页布局，包含查询区、表格区、分页区和操作区。
- 支持新增、编辑、删除等后台常用操作。
- 字段展示清晰，适合业务数据管理。
- 页面风格与系统其他模块一致，便于后续扩展。

**价值说明**

- 降低重复编码成本。
- 提升页面交付效率。
- 保持多模块之间的交互风格统一。
- 适合快速构建标准化业务管理界面。

### 4. 国际化页面

![国际化页面](./images/4%20i18n.png)

国际化页面展示了 GoAdmin 的多语言能力。界面切换到英文后，菜单、标题、表格和操作文案均同步更新。

**页面特点**

- 支持语言切换。
- 菜单与页面标题可同步翻译。
- 搜索条件、按钮和表头可本地化。
- 适合多语言团队与跨地域部署场景。

**价值说明**

- 为国际化部署提供基础能力。
- 提升系统在多语言环境中的可用性。
- 为后续扩展更多语言预留空间。

## 项目定位

GoAdmin 面向希望快速搭建管理后台的开发团队，重点解决以下问题：

- 后台资源分散、维护成本高。
- 业务页面重复开发、交付效率低。
- 权限、菜单、路由和页面之间缺少统一治理。
- 国际化能力不足，难以支持多语言场景。

通过模块化后端、标准化前端和生成式开发流程，GoAdmin 让后台系统具备更高的一致性和可扩展性。
